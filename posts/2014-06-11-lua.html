Несколько притч на тему языка для управления синтезатором

Posted on June 11, 2014 with tags: haskell, lua, синтезатор, hardware.

Почему lua не подходит для программирования синтезатора

  • Lua — тьюринг-полный язык программирования. Полнота по тьюрингу

Откат

Поскольку обмазать всю программу исключениями на тему «вот сейчас вот выключится электричество» вряд ли удастся, а сам скрипт-протокол за исключением дёрганья API синтезатора и перемешивания памяти в своей куче не делает ничего полезного и никак иначе не взаимодействует со внешним миром, то можно попробовать следующую уловку: логировать все результаты API вызовов синтезатора выполненных для этого скрипта и в случае восстановления после сбоя запустить скрипт, предоставив ему фейковое API. Если какой-то шутник не напишит на чистом лунном генератор случайных значений, то скрипт таким образом вернётся в последнюю залогированную точку. Впрочем есть нюансы.

Prolog

Основная работа, которая видимо полностью остаётся на совести скрипта-протокола это выбор лунок с реактивами в рантайме. Поскольку в процессе работы синтезатора реактивы могут не только убывать, но и появляться на столе, задача становится неразрешимой в статике. Поскольку читать и писать программу должны живые люди было бы здорово максимально повысить читаемость выходного кода. Есть идея написать для lua пролог-подобный edsl позволяющий по неким фактам представленным синтезатором составлять утверждения и на их основе искать лунки и прочие объекты.
